# Release 454 (15 Aug 2024)

## General

* Add support for retrieving lightweight query information from the `/v1/query`
  endpoint with the `pruned` query parameter. ([#20843](https://github.com/trinodb/trino/issues/20843))
* Add support for providing configuration for a secrets provider in a
  `secrets.toml` file in the main Trino configuration directory. ([#22633](https://github.com/trinodb/trino/issues/22633))
* Add support for an event listener plugin that publishes events to Kafka. ([#22426](https://github.com/trinodb/trino/issues/22426))
* Add temporary support for restoring unsafe pushdown behavior removed in
  version 445. This can be enabled with the `optimizer.allow-unsafe-pushdown`
  configuration property. When this mode is enabled, certain queries containing
  expressions that may fail for some inputs may fail at runtime. ([#22987](https://github.com/trinodb/trino/issues/22987))
* Improve performance of queries with multiple aggregations involving
  `DISTINCT`. ([#22355](https://github.com/trinodb/trino/issues/22355))
* {{breaking}} Improve query concurrency when memory is low by removing the
  `query.low-memory-killer.delay` configuration property. ([#22936](https://github.com/trinodb/trino/issues/22936))
* Avoid retrying failed filesystem operations when they will always fail. ([#22678](https://github.com/trinodb/trino/issues/22678))
* Fix failure for queries involving {func}`json_parse` and a cast to `array`,
  `map`, or `row`. ([#22840](https://github.com/trinodb/trino/issues/22840))
* Fix query failure when file-based network topology is configured with the
  `node-scheduler.network-topology.file` configuration property. ([#18793](https://github.com/trinodb/trino/issues/18793))

## Web UI

* Add information about which tables and UDFs have been referenced by a
  query. ([#20843](https://github.com/trinodb/trino/issues/20843))

## JDBC driver

* Add support for the `NO_GENERATED_KEYS` option in the
  `Connection.prepareStatement` method. ([#22100](https://github.com/trinodb/trino/issues/22100))
* Fix the Trino username incorrectly defaulting to the name of the user running
  the Trino process when no username is specified. ([#22901](https://github.com/trinodb/trino/issues/22901))

## BigQuery connector

* Add support for the `execute` procedure. ([#22692](https://github.com/trinodb/trino/issues/22692))

## Delta Lake connector

* Add S3 security mapping for the native S3 filesystem. ([#22559](https://github.com/trinodb/trino/issues/22559))
* Add support for writing
  [deletion vectors](https://docs.delta.io/latest/delta-deletion-vectors.html). ([#17063](https://github.com/trinodb/trino/issues/17063))
* Improve performance by caching table metadata to the metastore. ([#21463](https://github.com/trinodb/trino/issues/21463))
* {{breaking}} Rename metastore OpenTelemetry attributes from `trino.hive.*` to
  `trino.metastore.*`. ([#22866](https://github.com/trinodb/trino/issues/22866))

## Hive connector

* Add support for using the `hive.metastore-refresh-interval` configuration
  property when using Glue v2 Hive metastore. ([#22706](https://github.com/trinodb/trino/issues/22706))
* Add S3 security mapping for the native S3 filesystem. ([#22559](https://github.com/trinodb/trino/issues/22559))
* {{breaking}} Rename metastore OpenTelemetry attributes from `trino.hive.*` to
  `trino.metastore.*`. ([#22866](https://github.com/trinodb/trino/issues/22866))
* Add support for adjusting UTC-adjusted `INT64` timestamps to the timezone
  specified in the `hive.parquet.time-zone` property when reading Parquet files. ([#22577](https://github.com/trinodb/trino/issues/22577))
* Fix potential query failure caused by incorrect bucket column validation. ([#22261](https://github.com/trinodb/trino/issues/22261))
* Fix error when reading `INT96` timestamps in Parquet files generated by AWS
  Wrangler. ([#22854](https://github.com/trinodb/trino/issues/22854))

* Add S3 security mapping for the native S3 filesystem. ([#22559](https://github.com/trinodb/trino/issues/22559))
